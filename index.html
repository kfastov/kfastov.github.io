03 Private-work case studies

The work is private. The engineering is explainable.

These summaries separate platform context from my personal scope and intentionally omit strategy rules, private code, account data, and infrastructure details.

Rust · trading systems

From historical live execution to a cleaner successor

I designed, implemented, and operated a Rust funding-rate arbitrage system spanning exchange connectivity, streaming market data, state-machine-based order execution, position reconciliation, and deterministic backtesting.

Historical live execution ran on Binance USD-M and Hyperliquid from February to May 2026. It is currently paused while I rebuild the system from first principles as a strategy-agnostic engine with typed domain boundaries, capability-based connectors, and shared deterministic semantics for live and simulated execution.

Current status: the successor has foundational domain and connector work in place, but remains an early implementation—not a finished trading engine.
Personal scope
Architecture, implementation, production operation, incident analysis, and redesign.
Engineering themes
Explicit order state, reconciliation, deterministic replay, venue boundaries, and operational safety.

Go · blockchain data

APIs and indexers at Ankr

I worked on Ankr Advanced API, a blockchain-data platform spanning 13+ EVM networks and Solana. That number describes platform context; my contribution was concentrated in Go services for indexing, public APIs, RPC routing, and node access.

My work included JSON-RPC and gRPC boundaries, token and balance indexing, DEX-liquidity price routing, Geth/Erigon transaction tracing, node selection, request-path parallelization, and application deployment tooling.

Disclosure boundary: production repositories are not public. This summary contains no private code, schemas, endpoints, customer data, or internal screenshots.
Personal scope
Backend features, cross-service API contracts, performance work, debugging, and Helm deployment configuration.
Engineering themes
Idempotent indexing, backpressure, fan-out latency, heterogeneous RPC nodes, and multi-service delivery.

Diagnose partial-fill handling, cancel/replace races, local-versus-exchange state drift, reduce-only hazards, and incidents that resist reproduction.
